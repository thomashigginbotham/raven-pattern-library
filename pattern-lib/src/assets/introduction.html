Welcome! If you‘re reading this in your browser, then hopefully, you‘re set up and ready to go. Here‘s what you want to do next.

Next Steps

  1. Replace this introductory content with your own.
  2. Update the navigation.
  3. Add your own HTML and styles.

Replace Introductory Content

The major sections of the pattern library (Introduction, Base Styles, Components, and Pages) can be easily updated. Navigate to the /pattern-lib/src/assets directory and look for the appropriate HTML files to change.

Change the Navigation

The Components and Pages sections of the navigation start you off with some common elements, but your unique project will certainly require its own set of components and pages. Look in the appropriate sections of the /pattern-lib/src/assets/rpl-config.json file to customize the navigation as you see fit.

Update HTML, Styles, and Scripts

RPL comes with some basic HTML pages and style sheets. These are here to get you started and familiar with RPL’s structure.

HTML

In the html directory, you will see three pages: index.html, interior.html, and form.html. Replace their content or remove them as you see fit.

Under the html directory is a components directory. This is where you will place the components that will be used on the pages. Take a look at them to see how they correlate to the Components section of the pattern library.

Sass

The scss directory contains all of your style sheets. If you’re new to Sass, don’t feel intimidated. Take a look at the files that are there and play around with them to see what effect your changes make.

JavaScript

The js directory will house your JavaScript files. To keep things simple, you won’t find any sample JS code.
