So purple isn’t your thing? No worries, All the colors and branding within the pattern library are customizable. We make use of CSS variables to give you quick styling options, but you can take it even further and overwrite any CSS used by RPL.

Change the Logo and Heading

Chances are you’d rather use your own logo and name at the top of your pattern library. We make use of a JSON configuration file found at rpl-assets/rpl-config.json for these types of settings.

Change the title and logoUri values to something appropriate. An SVG image works best for the logo, and it should be 75×75 px for best results. However, the logo dimensions and its top/left offset can be adjusted with CSS variables (see below).

Add Custom CSS

RPL comes with a special style sheet that you can use to overwrite the CSS used by the pattern library. This gives you complete stylistic control over how RPL looks.

As with most of RPL’s customization, you’ll find the file in the rpl-assets directory. The file you want to update is rpl-styles.css.

Common CSS Variables

Name Description
--color-brand-primary The color used by the header at the top as well as by headings and links.
--color-brand-primary-50 A lighter version of the primary color.
--color-body The default text color (use your website’s color for best results).
--color-text-light Muted text (should be similar to --color-body but less noticeable).
--bg-color-body The background color (use your website’s background color for best results).
--color-links Color to use for hyperlinks.

For a full list of the available CSS variables, see rpl-src/src/styles.css
