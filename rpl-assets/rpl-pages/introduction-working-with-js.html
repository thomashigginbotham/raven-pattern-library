If your JavaScript needs are fairly lightweight, you can just add some JS files to the src/scripts folder, add a <script> tag to your HTML pages, and call it a day. However, most websites today are going to depend on a bit more than that, and you’ll probably want your components to make use of your scripts when viewed in RPL, not just on your HTML pages.

Script Entrypoint

If you’ve worked with Node or Webpack, you’re probably familiar with a script entrypoint. It’s a function that kicks off all the scripts that will run on the page. Lucky for you, we went ahead and created one for you. If you look in the src/scripts directory, you’ll see a file called main.js.

The main.js file has an entrypoint function called myScripts.initComponents(). This function is called automatically by RPL every time components are viewed within the pattern library. Outside of the pattern library, the function will be called as long as you include a <script> tag that references the main.js file (we do this for you out of the box).

If you don’t like our entrypoint, you can make your own by editing the initComponentsScript value in the rpl-config.json file. If you choose not to use the main.js file at all, make sure to reference your script files by updating the scriptUris array in the JSON file as well.

Using a Module Bundler

If you will be including a moderate amount of JavaScript, we advise using a module bundler like Webpack. The setup is beyond the scope of this guide, as it will involve changes to gulpfile.js to work with your specific implementation.
