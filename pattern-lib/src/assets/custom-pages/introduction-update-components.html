Update a Component

Your components are found in the /html/components directory. To make updates, you can simply open the HTML file and make your changes. Bing, bang boom!

Create a Component

The easiest way to create a new component is to duplicate an existing one from the /html/components directory and change the file name. The file name can be anything you like, and you can create subdirectories for your components to help organize things better.

Components have HTML comments at the top to allow you to give them a name and a summary. This information will appear above the component when viewed in the pattern library.

Adding a Component to the Main Menu

Before a component actually appears in the pattern library, you will need to tell RPL where it goes. By default, you will see that the components section of the main navigation has a few groups where components are placed (such as Page Structure or Lists). You should know that these groups are customizable, and we expect you to modify them to your own needs. To do this, you will need to modify the /pattern-lib/src/assets/rpl-config.json file.

You may already be aware of the navigation key in the RPL config JSON file. This is where you will configure the groups available to your components. Give each group a name and a URI. The URI can be anything as long as it begins with components/.

Now that you’ve decided on which group to use for your component, you need to add it to that group. In the same JSON file, you should see a components key. It contains an array of component groups, and each group has a list value that controls which components appear there. First, ensure that any changes that you made to the groups in the navigation are reflected here, and then add your component to the list.

JSON Component Settings

Key Value
uriKey Matches the component group URI value from the navigation without the components/ part.
heading Appears at the top of the page when the component group is viewed.
description Appears below the heading at the top of the page.
list A list of the components to add to the page. Each item in the list has a title (the text to use in the menu) and a uri (the component file path relataive to the /html directory and without the .html extension.

Including a Component on a Page or from another Component

If you had to type out the HTML every time you wanted to use a component, that would be a problem. Fortunately, that’s not the case. To include a component on one of your HTML pages (or from within another component), add a comment in the following format:

<!-- build:include path/to/component.html --><!-- /build -->

The path to your component is relative to the file where it is being included. For example, if you have a component at /html/components/parent-component.html and you want to include a component from /html/components/children/child-component.html, you would use the following:

<!-- build:include children/child-component.html --><!-- /build -->
