Out of the box, RPL generates a bundle.js file using any TypeScript files it finds in the /src/scripts directory. Webpack and Babel are used behind the scenes, allowing you to focus on writing clean, modular code.

TypeScript adds type support to JavaScript. You can learn more at typescriptlang.org. If you prefer vanilla JavaScript, simply create your own main.js file in the src/scripts directory, and remove the main.ts file.

Script Entrypoint

If you’ve worked with Node or Webpack, you’re probably familiar with a script entrypoint. It’s a function that kicks off all the scripts that will run on the page. Lucky for you, we went ahead and created one for you. If you look in the src/scripts directory, you’ll see a file called main.ts.

The main.ts file exports a class with an initComponents() method. This method is called automatically by RPL every time components are viewed within the pattern library. Outside of the pattern library, the method will be called on every page load.

If you don’t like our entrypoint, you can make your own by editing the initComponentsScript value in the rpl-config.json file. If you choose not to use the main.ts file at all, make sure to reference your script files by updating the scriptUris array in the JSON file as well.

Modules

A module is simply a variable, function, or class that is exported from a JavaScript file so that it can be used in another file. We start you off with a sample module that exports a class with a single method in it (/src/scripts/sample-module.ts). Edit or delete the file, and start writing your own modules.
